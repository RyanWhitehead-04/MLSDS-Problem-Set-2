# <p align = center> How effective is BERTopic at classifying Community Ownership Fund Projects? (1,500 words) </p>

This project is intended to fulfil the requirements of Problem Set 2 of the [Machine Learning for Social Data Science](https://www.exeter.ac.uk/study/studyinformation/modules/info/?moduleCode=SSIM916&ay=2025/6&sys=0) module at the University of Exeter

It applies BERTopic, an unsupervised machine learning technique, to categorise and analyse the types of projects funded by the UK's Community Ownership Fund (COF)

&nbsp;

## :dart: Project Aim
The project aims to answer three questions:
1.  Can BERTopic generate coherent and distinct topics for COF projects?
2.  How accurately does BERTopic assign labels to COF projects compared to manual classification?
3.	Do the topics generated by BERTopic provide substantively meaningful insights into the nature and purpose of COF projects?

&nbsp;

## :bar_chart: Dataset
**Source:** A text corpus built from short document summaries from a range of sources

**Sample Size:** 338 Projects  

**Unit of Analysis:** Individual Projects

&nbsp;

## :lock: Ethics
* Uses publicly available data
* Minimal risk of harm to projects

&nbsp;

## :robot: Models Used
[BERTopic](https://maartengr.github.io/BERTopic/index.html) is a machine learning technique designed to uncover latent topics from bodies of text

The key steps of setting up the model and parameters for this project include:
1. Emeddings (all-MiniLM-L6-v2)
2. Dimensionality Reduction (n_neighbors = 5, min_dist = 0.00)
3. Clustering (HDBSCAN: min_cluster_size = 30, min_samples = 3)
4. Vectorisation (Detecting unigrams and bigrams)
5. Representation model (KeyBERTInspired)

*Alternative embedding models were also tested to evaluate robustness; these are labelled as solution 2 (all-MiniLM-L12-v2) and solution 3 (all-Mpnet-base-v2) in the code folder*

&nbsp;

## :chart_with_upwards_trend: Evaluation
Evaluation of model performance was broken down into 3 layers:

* Topic evaluation
  * Topic Coherence
  * Topic Diversity
  * Relationship between Topics

* Label evaluation
  * Precision
  * Recall
  * F1 Score
  * Accuracy
  
* Substantive Evaluation
  * What sort of projects have been detected?
  * What does this tell us about COF projects? 

&nbsp;

## :mag_right: Key Findings
:one: **Model Performance**
* BERTopic identified 5 common topics:
    * Community Spaces
    * Leisure and Fitness
    * Economic
    * Heritage, Culture, and Tourism
    * Community Pubs
* BERTopic generated coherent and interpretable topics, with strong coherence in some areas, particularly projects related to community pubs and leisure and fitness
* However, the model also showed some difficulty in distinguishing between closely related topics, such as community spaces and economic, and heritage, culture, and tourism, and community pubs.
* When compared to human classification, the model had moderate performance, capturing broad patterns but struggling with precise classifications, particularly between closely related topics

:two: **Substantive Performance**
* COF projects are focused on maintaining community assets and local services
* COF projects do not focus on driving structural economic changes 

&nbsp;

## :warning: Limitations
* This project only focuses on COF projects and therefore cannot be generalised to other funding streams
* The model struggles with the precise classification of projects, particularly topics which are closely related

&nbsp;

## :rocket: Future Improvements
* **Refine Preprocessing:** Remove location- and policy-specific terms and standardise project descriptions to improve model performance
* **Zero-shot Learning:** Future models could explore zero-shot classification to see if they can generate accurate labels from human-provided categories
* **Expanding the Text Corpus:** Include projects from other Levelling Up funding streams to improve generalisability
